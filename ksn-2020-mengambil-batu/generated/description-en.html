Description

Pak Dengklek asks you to play with him. In the beginning, there are $K$ piles of stones numbered from $1$ to $K$. The $i$-th pile has $A_i$ stones.

The players take turns alternately. The first turn is played by you. For each turn, a player takes at least one stone from one of the piles as long as the number of stones does not exceed $N$ and there are still enough stones in that pile. If a player cannot make a valid move in their turn, then that player loses the game and the game is finished.

Since you are a competitive person, you do not want to lose against Pak Dengklek. Therefore, you want to make a program that can beat Pak Dengklek.

Problem Type Information

This is an "interactive" problem. In this problem, you will interact with the judge's program through standard input (stdin) and standard output (stdout). See the following interaction format carefully.

Interaction Format

In the beginning, the value of $K$, $N$, and all values of $A_i$ will be given from the standard input with the following format: 

K N
A1 A2 ... AK

Then, for each turn, a player can take $x$ stones from the $i$-th piles with the following format: 

i x

If Pak Dengklek cannot make any valid move, then you will be given an input in the following format: 

-1 -1

Interaction is finished with this input and your program is considered as correct for this test case.

Subtasks

For all subtasks:

  • $1 \leq K \leq 2$
  • $1 \leq A_i \leq 10^4$, for $1 \leq i \leq K$
  • $1 \leq N \leq 10^4$
  • There exists a strategy that can make the first turn player win the game

Subtask 1 (0 points)

  • $K = 1$

Subtask 2 (0 points)

  • $K = 2$
  • $1 \leq A_i \leq 100$
  • $1 \leq N \leq 100$

Subtask 3 (0 points)

  • $K = 2$

Warning

Always flush after each output.

  • Pascal:
    flush(output);
  • C/C++:
    • for stdio.h (cstdio) library:
      fflush(stdout);
    • for iostream library:
      std::cout << std::flush;
