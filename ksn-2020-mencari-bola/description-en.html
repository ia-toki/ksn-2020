Description

Pak Dengklek has $N$ boxes numbered from $1$ to $N$. Between these $N$ boxes, there exist exactly $K$ boxes which contain a ball. You want to find out which boxes contain a ball by asking several questions to Pak Dengklek.

In one question, you ask several sets $S_1, S_2, \dots, S_x$, where $x$ is the number of sets you give in that question. Each set consists of up to $N$ elements, which are distinct integers between $1$ and $N$. This question will be answered with $x$ answers. The $i$-th answer is based on set $S_i$ and will be:

  • YA, if there is at least one box that contains a ball and its number is an element of $S_i$; or
  • TIDAK, otherwise

Define $\min(|S_i|)$ as a function which returns the minimum value of $|S_1|, |S_2|, \dots, |S_x|$, where the notation $|S|$ represents the cardinality of the set $S$. The cost of this question is $\frac{1}{\min(|S_i|)}$. You want to minimize the total cost of all questions you give until you can correctly guess all of the boxes which contain a ball. Your score will depend on the total cost, using a formula that will be explained in the Scoring section.

Problem Type Information

This is an "interactive" problem. In this problem, you will interact with the judge's program through standard input (stdin) and standard output (stdout). See the following interaction format carefully.

Interaction Format

In the beginning, the values of $N$ and $K$ will be given from standard input in the following format: 

N K

Then, for each question, you can output to standard output in the following format: 

? x
C1 S1,1 S1,2 ... S1,C1
C2 S2,1 S2,2 ... S2,C2
.
.
.
Cx Sx,1 Sx,2 ... Sx,Cx

This means you give $x$ sets numbered from $1$ to $x$. The set $S_i$ has $C_i$ elements $S_{i,1}, S_{i,2}, \dots, S_{i,C_i}$ which are distinct integers between $1$ and $N$.

For each question, Pak Dengklek will give answers from standard input in the following format: 

R1
R2
.
.
.
Rx

where each $R_i$ can be either YA or TIDAK as an answer for the set $S_i$ following the answer rules explained in the Description section.

In the end, your answer has to be output to standard output in the following format: 

! i'1 i'2 ... i'K

This means you guess $K$ boxes that boxes with number $i'_1, i'_2, \dots, i'_K$ each contains a ball. Your answer will be considered as correct if all $K$ guessed boxes indeed contain a ball, and you will get score using a formula that will be explained in the Scoring section.

Explanation of Interaction Sample 1

The first question costs $\frac{1}{1}$, and the second question costs $\frac{1}{2}$. The total costs for all questions are $\frac{3}{2}$. Using the formula given in the Scoring section, the score you get from this interaction sample is $5$.

Scoring

There are restrictions on the number of questions you can ask Pak Dengklek:

  • The number of sets you give in a question must not exceed $10$
  • The total number of sets you give in all questions must not exceed $N$

If any of the restictions above is violated, you will receive score $0$.

The formula used to determine your score depends on the value of $K$. Suppose that $P$ is the total cost from all of your questions. For $K = 1$, the score that you get is:

Condition Points
$N \leq P$$0$
$1 \leq P \lt N$$\lfloor 10 \times (1 - \sqrt{\frac{P - 1}{N - 1}} ) \rfloor$
$\frac{2}{N} \lt P \lt 1$$\lfloor 30 - 20 \times (1 - \frac{2}{N \times P})^{0.8} \rfloor$
$P \leq \frac{2}{N}$$30$

Meanwhile for $K = 2$, the score that you get is:

Condition Points
$N \leq P$$0$
$1 \leq P \lt N$$\lfloor 20 \times (1 - \sqrt{\frac{P - 1}{N - 1}} ) \rfloor$
$\frac{4}{N} \lt P \lt 1$$\lfloor 70 - 50 \times (1 - \frac{4}{N \times P})^{0.8} \rfloor$
$P \leq \frac{4}{N}$$70$

Subtasks

For all subtasks:

  • $N = 2^{10}$
  • $1 \leq K \leq 2$
  • $K = 1$
  • $K = 2$

Warning

Always flush after each output.

  • Pascal:
    flush(output);
  • C/C++:
    • for stdio.h (cstdio) library:
      fflush(stdout);
    • for iostream library:
      std::cout << std::flush;
